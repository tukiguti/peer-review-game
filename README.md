# 査読ゲーム (peer-review-game)

設定した1〜5枚の研究カードを引き、その場ででっち上げた研究内容を語り、他プレイヤーが「査読が通るか」を秘密投票する対面パーティゲーム。

> 「量子コンピューティング」×「エスノグラフィ調査」×「被験者は3人まで」
> ——このカード構成で、それらしい研究を語れ。過半数が Accept すれば採択。

- **形態**: Webアプリ。起動時に**オフライン（端末1台を囲んで対面・サーバ不要）**／**オンライン（各自スマホ・準備中）**を選ぶ
- **人数**: 3〜8人
- **1プレイ**: 5人 × 2周でおよそ25分

## 遊び方

1. **抽選** — 1〜5リールのスロットが回り、設定した種類のカードが決まる
2. **発表** — 発表者が60秒で研究内容を語る（制約カードの無茶ぶりは無視できない）
3. **査読** — 発表者以外が `Accept` / `Reject` を投票。既定は**せーの（全員同時公開）**、設定で**パスプレイ（1人ずつ秘密投票）**にも切替可
4. **判定** — 過半数 Accept で採択。発表者に +2点（満場一致なら +3点）
5. 査読者も、**多数派側に投票していれば +1点**

カード構成は各スロットを「分野」「手法」「制約」「新規性」から自由に選び、さらに「おまかせ」「真面目」「ネタ」を個別に指定できます。たとえば「分野・真面目／分野・ネタ／制約・ネタ」のような混在も可能です。既定は「査読4枚（分野・手法・制約・新規性）」で、ほかに「標準3枚」「分野×3」「ライト2枚」「盛り盛り5枚」のプリセットもあります。

Accept の目安は「全カードを使った」「カード同士のつなぎ方に筋が通った」「制約への工夫に納得できた」の3項目です。最終結果では総得点1位を「学会MVP」、発表で得た点が最多の人を「最優秀論文賞」として分けて表彰します。

カードは汎用・ソフトウェア工学・セキュリティ・ファッションの4ジャンル、真面目寄せ・ネタ寄せの両方を収録しています（分野・手法・制約・新規性の4デッキ、全339枚）。

## 開発

```bash
npm install
npm run dev      # http://localhost:5173
npm run test     # 得点計算・抽選ロジックのユニットテスト
npm run build
```

公開前の確認項目は [docs/spec.md](docs/spec.md) にまとめています。変更時は `npm test` と `npm run build` の両方を通してください。

デプロイ: main への push で自動配信。本番は **Cloudflare Worker（静的アセット・無料）** → https://review-game.tukiguti.com/
（GitHub Pages のミラーも維持: https://tukiguti.github.io/peer-review-game/ ／ `base` はビルド時に自動で出し分け）
